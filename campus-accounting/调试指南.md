# 微信小程序真机测试调试指南

## 问题描述

在微信开发者工具中调试正常，但在真机上无法正常测试，可能出现的问题包括：
- 无法启动
- 白屏
- 网络请求失败
- 图片无法加载
- 功能异常等

## 常见问题与解决方案

### 1. 网络请求问题

#### 问题1：request:fail 域名不合法
**现象**：真机上请求接口失败，提示域名不合法

**解决方案**：
1. 登录微信公众平台
2. 进入"开发管理" → "开发设置"
3. 在"服务器域名"中添加合法的域名
4. 域名需要是 HTTPS 协议
5. 保存后重新上传版本

**临时解决方法（仅测试）**：
在微信开发者工具中，点击"详情" → "本地设置"，勾选"不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书"

#### 问题2：网络请求超时
**现象**：网络请求长时间无响应或超时

**解决方案**：
1. 检查服务器是否正常运行
2. 优化接口响应时间
3. 增加请求超时时间设置
4. 检查网络环境是否稳定

### 2. 图片加载问题

#### 问题：图片无法加载
**现象**：真机上图片不显示或显示破裂图标

**解决方案**：
1. 检查图片链接是否正确
2. 确保图片是 HTTPS 协议
3. 检查图片格式是否支持（推荐使用 JPG 或 PNG）
4. 检查图片大小是否过大（单个图片不建议超过 2MB）
5. 图片链接不要包含中文或特殊字符

### 3. 云开发问题

#### 问题1：云开发环境初始化失败
**现象**：无法访问云开发数据，提示环境错误

**解决方案**：
1. 检查 app.js 中云开发环境 ID 配置
2. 在微信开发者工具中，点击"云开发"查看环境状态
3. 确保云开发环境已开通并正常运行
4. 检查环境 ID 是否正确

**修复方法**：
在 app.js 中修改云开发初始化代码：
```javascript
// 使用自动获取当前环境
wx.cloud.init({
  env: wx.cloud.DYNAMIC_CURRENT_ENV,
  traceUser: true
})
```

#### 问题2：云数据库无权限
**现象**：无法读取或写入云数据库

**解决方案**：
1. 检查数据库集合权限设置
2. 在微信开发者工具中，点击"云开发" → "数据库"
3. 选择对应的集合，点击"权限设置"
4. 开发阶段可以设置为"所有用户可读，仅创建者可读写"

### 4. 权限问题

#### 问题：缺少权限配置
**现象**：某些功能在真机上无法使用

**解决方案**：
1. 检查 app.json 中的权限配置
2. 确保所需的权限已在 manifest 文件中声明
3. 在微信公众平台"设置" → "功能设置"中配置

**常见需要配置的权限**：
- `scope.userInfo`：获取用户信息
- `scope.address`：获取用户地址
- `scope.phoneNumber`：获取用户手机号码

### 5. 存储问题

#### 问题：本地存储数据丢失
**现象**：存储的数据在真机上无法读取

**解决方案**：
1. 检查存储路径是否正确
2. 使用 `wx.getStorageSync()` 和 `wx.setStorageSync()` 进行调试
3. 确保存储的数据格式正确
4. 检查存储大小是否超过限制（单个 key 最大 1MB，总大小 10MB）

### 6. 其他常见问题

#### 问题1：白屏
**现象**：真机上显示白屏，无任何内容

**解决方案**：
1. 检查页面路径配置是否正确
2. 检查是否有语法错误导致页面无法加载
3. 检查页面数据初始化是否有误

#### 问题2：页面跳转失败
**现象**：页面跳转时提示路径不存在

**解决方案**：
1. 检查 app.json 中 pages 配置是否正确
2. 检查跳转路径是否拼写错误
3. 确保页面文件已正确创建

## 调试方法

### 1. 真机调试

1. 点击微信开发者工具中的"真机调试"
2. 选择"普通调试"或"基础库调试"
3. 在手机上扫码打开小程序
4. 可以实时查看 console 输出和网络请求

### 2. 添加调试信息

在关键位置添加 console.log：

```javascript
// 在 app.js 中添加全局错误监听
App({
  onError: function(msg) {
    console.error('全局错误:', msg)
  },
  onUnhandledRejection: function(e) {
    console.error('未处理的 Promise 拒绝:', e)
  }
})

// 在页面中添加错误监听
Page({
  onLoad: function() {
    try {
      // 你的代码
    } catch (error) {
      console.error('页面加载错误:', error)
    }
  }
})
```

### 3. 使用本地存储模拟数据

```javascript
// 在 app.js 中添加模拟数据
globalData: {
  useLocalStorage: false,
  // 如果云开发失败，使用本地存储数据
  mockData: {
    records: [
      { id: 1, date: '2023-12-01', category: '餐饮', amount: 20, type: 'expense' },
      { id: 2, date: '2023-12-02', category: '兼职', amount: 100, type: 'income' }
    ]
  }
}

// 在页面中使用
Page({
  data: {
    records: []
  },
  onLoad: function() {
    const app = getApp()
    if (app.globalData.useLocalStorage) {
      this.setData({
        records: app.globalData.mockData.records
      })
    } else {
      // 正常从云数据库获取数据
      this.loadRecords()
    }
  },
  loadRecords: function() {
    // 你的数据获取逻辑
  }
})
```

## 测试步骤

1. **基础功能测试**：
   - 首页显示
   - 记录添加
   - 记录查看
   - 预算设置

2. **高级功能测试**：
   - 分类筛选
   - 图表展示
   - 预警功能
   - 安全检测

3. **性能测试**：
   - 页面加载时间
   - 数据加载速度
   - 内存消耗

## 总结

遇到真机测试问题时，不要慌张，按照以下步骤分析：

1. 首先使用微信开发者工具真机调试功能定位问题
2. 查看 console 输出的错误信息
3. 根据错误信息搜索对应的解决方案
4. 尝试修复后重新上传测试
5. 如果问题复杂，可以使用分步测试方法

如果以上方法都无法解决问题，请检查：
- 微信小程序基础库版本是否兼容
- 手机系统版本是否支持
- 是否有其他程序干扰

祝你测试顺利！
